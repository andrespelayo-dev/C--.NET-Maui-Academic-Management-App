# .NET MAUI Academic Management Application

A cross-platform academic management application developed using C# and .NET MAUI as part of Western Governors University's Software Engineering program.

## Features

- Manage courses and assessments
- Track academic progress
- Local database storage
- Mobile-friendly interface
- Notifications and reminders

## Technologies Used

- C#
- .NET MAUI
- SQLite
- MVVM Architecture
- CommunityToolkit.Mvvm

## Skills Demonstrated

- Object-Oriented Programming (OOP)
- Mobile Application Development
- Database Integration
- MVVM Design Pattern
- Software Development Lifecycle (SDLC)

## Project Structure

- Models - Application data models
- ViewModels - Presentation and business logic
- Views - User interface pages
- Services - Database and application services
- Resources - Images and application assets

## About

This project was created as part of my Software Engineering degree and demonstrates mobile application development using C#, .NET MAUI, and SQLite. It was built to help users manage academic terms, courses, assessments, and reminders in a single mobile interface.

The application uses MVVM principles and data binding to separate interface logic from application behavior. It also integrates a SQLite database for persistent local storage and supports full CRUD functionality for terms, courses, and assessments.

In addition to the features shown in the screenshots, this project includes validation and business rules, notifications and reminders, and iterative debugging and testing throughout development. It also reflects practical SDLC concepts, including planning, implementation, testing, and refinement of a working mobile application.

## Development Highlights

- Built using MVVM architecture for separation of concerns
- Integrated SQLite for persistent local data storage
- Implemented full CRUD workflows across related entities
- Added validation logic and business rules for data integrity
- Implemented local notifications for reminders and deadlines
- Used debugging and repeated testing to improve reliability

## Challenges and Solutions

A key challenge in this project was managing related entities such as terms, courses, and assessments while keeping the workflow simple for the user. I addressed this by organizing responsibilities across models, viewmodels, and services, which improved maintainability and navigation flow.
Another challenge was ensuring data reliability. I implemented validation checks, tested edge cases, and used emulator-based debugging to resolve issues involving persistence, reminders, and input handling.
